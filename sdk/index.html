Installation

# Install from PyPI pip install s4-ledger # Or clone from GitHub git clone https://github.com/s4ledger/s4-ledger.git cd s4-ledger pip install -r requirements.txt

Requirements: Python 3.10+, xrpl-py, cryptography, requests

Quick Start

from s4_sdk import S4SDK # Initialize the SDK sdk = S4SDK( wallet_seed="sEdxxxxxxxxxxxxxxx", # Your XRPL wallet seed api_key="your_api_key", testnet=False # Use mainnet ) # Anchor a defense record result = sdk.anchor_record( record_text="Part received: NSN 5340-01-123-4567 Qty 50", wallet_seed="sEdxxxxxxxxxxxxxxx", record_type="SUPPLY_CHAIN" ) print(result["hash"]) # SHA-256 hash anchored on XRPL print(result["explorer_url"]) # Link to XRPL explorer # Verify a record against the chain verification = sdk.verify_against_chain( record_text="Part received: NSN 5340-01-123-4567 Qty 50", tx_hash=result["tx_results"]["fee_tx"]["hash"] ) print(verification["status"]) # "MATCH" — verified!

Authentication

The SDK uses two forms of authentication:

  • API Key — For S4 Ledger platform features (subscription validation, API access)
  • XRPL Wallet Seed — For blockchain operations (anchoring, fee payments). Your seed never leaves your local environment.
Security: Never expose your wallet seed in client-side code or version control. Use environment variables.

Core Functions

CORE anchor_record()

sdk.anchor_record(record_text, wallet_seed=None, encrypt_first=False, fiat_mode=False, gateway_issuer=None, record_type=None)

Full anchoring workflow: optionally encrypt, hash, and store on XRPL with 0.01 SLS micro-fee. The central function for creating immutable records.

ParameterTypeDescription
record_textstrThe record content to hash and anchor
wallet_seedstrXRPL wallet seed for signing transactions
encrypt_firstboolEncrypt record before hashing (for CUI/FOUO data)
fiat_modeboolUse USD-to-SLS conversion instead of direct SLS payment
record_typestrClassification: SUPPLY_CHAIN, MAINTENANCE, CUSTODY_TRANSFER, etc.
Returns: dict with hash, tx_results, record_type, explorer_url

UTIL create_record_hash()

sdk.create_record_hash(record_text)

Create a SHA-256 hash of any record text for later on-chain verification.

ParameterTypeDescription
record_textstrRecord content to hash
Returns: str — 64-character SHA-256 hex digest

VERIFY verify_against_chain()

sdk.verify_against_chain(record_text, tx_hash=None, expected_hash=None)

Verify record integrity against the XRPL. Recomputes SHA-256 and compares to the hash stored in the transaction memo.

ParameterTypeDescription
record_textstrRecord content to verify
tx_hashstrXRPL transaction hash to look up
expected_hashstrOr compare against a known hash directly
Returns: dict with verified, status (MATCH / MISMATCH / NOT_FOUND), explorer_url, tamper_detected

CORE correct_record()

sdk.correct_record(corrected_text, original_tx_hash, wallet_seed=None, reason="", record_type=None)

Re-anchor a corrected record with a supersession link. Both original and correction stay on-chain forever for full audit trail.

ParameterTypeDescription
corrected_textstrUpdated record content
original_tx_hashstrTX hash of the original (incorrect) record
reasonstrReason for the correction
Returns: dict with corrected_hash, correction_tx, supersedes, timestamp

sdk.setup_trust_line(wallet_seed, limit="1000000")

One-time setup of the SLS trust line so a wallet can hold and use SLS tokens. Required before any anchoring.

ParameterTypeDescription

XRPL store_hash_with_sls_fee()

sdk.store_hash_with_sls_fee(hash_value, wallet_seed, fee_sls="0.01", destination=None, record_type=None)

Low-level: pay 0.01 SLS micro-fee and store a hash in an XRPL transaction memo. Called internally by anchor_record().

ParameterTypeDescription
hash_valuestrSHA-256 hash to store in memo
wallet_seedstrXRPL wallet seed for signing
fee_slsstrSLS fee amount (default 0.01)
record_typestrRecord type tag for the memo
Returns: dict with fee_tx, memo_tx, explorer_url

XRPL wallet_from_seed()

sdk.wallet_from_seed(seed)

Create an XRPL Wallet object from a family seed. Auto-detects ED25519 seeds (starting with sEd) and falls back to secp256k1.

Returns: xrpl.wallet.Wallet

Encryption

CRYPTO encrypt_data() / decrypt_data()

sdk.encrypt_data(data: str) -> str sdk.decrypt_data(encrypted_data: str) -> str

Encrypt/decrypt sensitive data using Fernet symmetric encryption (AES-128-CBC + HMAC-SHA256). Use for CUI/FOUO content before hashing so the hash reveals nothing about the original data.

# Encrypt-then-anchor workflow encrypted = sdk.encrypt_data("CUI: Propulsion system deficiency report") result = sdk.anchor_record(encrypted, wallet_seed="sEdXXX") # Later, decrypt original = sdk.decrypt_data(encrypted) # Original text restored

ILS Analysis Tools

Integrated Logistics Support tools per MIL-STD-1390D, DoDI 4245.14, DoD 5000.73, FAR 46.7, and DFARS 246.7.

sdk.calculate_readiness(mtbf, mttr, mldt=0)

Calculate Operational Availability (Ao) and RAM metrics per MIL-STD-1390D.

ParameterTypeDescription
mtbffloatMean Time Between Failures (hours)
mttrfloatMean Time To Repair (hours)
mldtfloatMean Logistics Delay Time (hours)
Returns: dict with Ao, Ai, failure_rate, annual_failures, grade

ILS check_dmsms()

sdk.check_dmsms(nsn_list: list[str])

Check NSNs for Diminishing Manufacturing Sources & Material Shortages risk per DoDI 4245.14.

Returns: dict with per-NSN risk ratings, overall risk score, recommended actions

sdk.lookup_nsn(nsn: str)

Look up a National Stock Number in the Federal Logistics system. Returns FSC, NIIN, nomenclature, status, and hash.

Returns: dict with fsc, niin, nomenclature, status, hash

sdk.calculate_roi(programs=5, ftes=8, rate=145.0, license_cost=120000.0, audit_cost=250000.0, error_cost=8500.0, incidents=35)

Calculate ROI for S4 Ledger deployment. Estimates labor savings (65%), error reduction (90%), audit savings (70%).

ParameterTypeDescription

license_costfloatCurrent legacy system licensing costs
audit_costfloatCurrent annual audit costs

Returns: dict with net_benefit, roi_pct, payback_months, five_year_projection

Estimate total ownership cost per DoD 5000.73 / MIL-STD-881F. Includes acquisition, sustainment, DMSMS, and tech refresh costs.

Returns: dict with cost breakdowns, annual sustainment, total lifecycle cost

IMPORT import_json()

IMPORT import_and_anchor()

sdk.import_and_anchor(file_text, source_system, file_format="csv", wallet_seed=None, record_type=None, anchor=True)

Full pipeline: parse file (CSV/XML/JSON), hash each record, and anchor to XRPL. One function to import and immutably verify an entire dataset.

# Import and anchor a GCSS-Army CSV export with open("gcss_export.csv") as f: results = sdk.import_and_anchor( file_text=f.read(), source_system="gcss-army", file_format="csv", wallet_seed="sEdXXX", record_type="SUPPLY_CHAIN" ) print(f"Anchored {len(results)} records")

CLI Reference

s4-anchor CLI (12 Commands)

The embedded CLI accessed via python s4_sdk.py [command]:

# Anchor a record python s4_sdk.py anchor --record "Part received NSN 1234" --seed "sEdXXX" --type SUPPLY_CHAIN # Hash only (no blockchain) python s4_sdk.py hash --record "Maintenance completed" # Verify record python s4_sdk.py verify --record "Part received" --type expected:abc123... # SDK status python s4_sdk.py status # ILS commands python s4_sdk.py readiness --record "500,4,2" # MTBF,MTTR,MLDT python s4_sdk.py dmsms --record "5340-01-123-4567,5340-01-234-5678" python s4_sdk.py roi python s4_sdk.py lifecycle python s4_sdk.py warranty python s4_sdk.py action-items python s4_sdk.py calendar python s4_sdk.py provisioning

s4-cli (3 Commands)

Standalone CLI via python s4_cli.py [command]:

# Anchor with optional encryption python s4_cli.py anchor "CUI record content" --seed "sEdXXX" --encrypt --type MAINTENANCE # Verify against expected hash python s4_cli.py verify "Part received NSN 1234" --expected abc123def456... # Quick hash python s4_cli.py hash "Record content" # Pipe from stdin cat document.txt | python s4_cli.py anchor - --seed "sEdXXX"

REST API Endpoints

S4 Ledger also provides a REST API at https://s4ledger.com/api/. Full specification: OpenAPI JSON.

POST /api/anchor
Hash and anchor a record to XRPL
POST /api/hash
Create SHA-256 hash of record
POST /api/verify
Verify record against chain
GET /api/health
Check API and chain health
POST /api/wallet/provision
Create and fund new wallet
POST /api/wallet/buy-sls
Purchase SLS via subscription (USD→XRP→SLS DEX)
GET /api/wallet/balance
Check SLS and XRP balances
