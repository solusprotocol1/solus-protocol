Frequently Asked Questions

Common questions about S4 Ledger, security, and integration

Does S4 Ledger store classified or sensitive data on the blockchain? +
Absolutely not. S4 Ledger only anchors SHA-256 cryptographic hashes — mathematical fingerprints — to the XRPL. No actual documents, part numbers, serial numbers, PII, or classified information ever touches the blockchain. The hash is a one-way function: you cannot reverse-engineer the original data from it.
How does this integrate with existing DoW systems? +
S4 Ledger doesn't replace 3-M, GCSS-Army, DPAS, or any existing system. It adds an immutable verification layer on top. Via our REST API and Python SDK, any system that can make an HTTP call or run a Python script can anchor and verify records. It's designed to sit alongside, not replace, your current logistics infrastructure.
What is $SLS and why is a token needed? +
$SLS (Secure Logistics Standard) is a utility token on the XRP Ledger. It powers the micro-transaction fee (~$0.01) for each anchor. The token ensures the network is self-sustaining and spam-resistant. Organizations can pay in USD — our fiat mode auto-converts to $SLS behind the scenes so users never need to interact with cryptocurrency directly.
Is this NIST 800-171 / CMMC compliant? +
S4 Ledger is architected to support NIST 800-171 compliance. Since no CUI or sensitive data is stored on-chain (only hashes), the attack surface is minimal. Our roadmap includes formal NIST 800-171 certification and a FedRAMP pathway for government deployment.
Why the XRP Ledger instead of Ethereum or a private blockchain? +
XRPL settles transactions in 3-5 seconds at ~$0.001 per transaction. Ethereum costs $5-50+ per transaction. Private blockchains defeat the purpose — they're controlled by a single entity and aren't independently verifiable. XRPL provides the speed, cost, and neutrality required for defense-grade audit trails.
How is S4 Ledger funded? +
S4 Ledger is self-funded with a live $SLS token on XRPL mainnet, backed by a 100M-token supply with structured vesting. We are pursuing IRAD funding, SBIR/STTR applications, and strategic partnerships with defense organizations. Contact us to discuss investment or partnership opportunities.
Can my organization try S4 Ledger before committing? +
Yes. Our Pilot tier is free during beta. You'll get SDK access, up to 1,000 anchors, and a basic dashboard. This lets you test the technology on real (non-sensitive) logistics data before subscribing. Contact us to get started.
What ILS tools are included? +
S4 Ledger includes 19 interactive ILS Workspace tools: Gap Analysis, DMSMS Tracker, Readiness Calculator, Parts Cross-Reference, ROI Calculator, Lifecycle Cost Estimator, Warranty Tracker, Action Items & Task Tracker, Audit Record Vault, Defense Document Library, Compliance Scorecard, Provisioning & PTD Manager, AI Supply Chain Risk Engine, Audit Report Generator, Contract Lifecycle Management, Digital Thread / Configuration Bridge, Predictive Maintenance AI, Conversational AI Agent, and Defense Database Import. All tools share data, sync automatically, and anchor results to the XRPL.
Can I use my own platform, hull number, or program office? +
Yes. Every tool that uses a platform selector includes 500+ pre-loaded defense platforms across all branches — plus a "Custom Program" option for entering any platform, vessel, or aircraft not yet in the database. You can also input real hull numbers, tail numbers, designations, program offices, and contract numbers. This ensures S4 Ledger works with your actual program data, not just demo samples.
What is the SDK Playground? +
The SDK Playground is an interactive sandbox where you can test all 21 Python SDK functions directly in your browser with live API calls. It includes a platform/program selector with 500+ platforms, hull/designation input, program office input, and contract number input — so you can test SDK functions against specific programs before writing any code.
How does the Provisioning & PTD Manager compare to ICAPS? +
S4 Ledger's Provisioning & PTD Manager replaces ICAPS with significant improvements. ICAPS is limited to Navy/USMC, uses a legacy interface, and has no audit trail. S4 Ledger supports all branches, provides Provisioning Technical Documentation per MIL-STD-1552, Allowance Parts List generation, NSN cataloging via FCS, and blockchain-verified provisioning decisions — all with an immutable XRPL audit trail.
What does the Compliance Scorecard measure? +
The Compliance Scorecard auto-calculates your compliance posture across 6 frameworks: CMMC Level 2, NIST SP 800-171 (110 controls), DFARS 252.204-7012, FAR 46 Quality, MIL-STD-1388 ILS, and DoDI 4245.15 DMSMS. Your score is calculated from real workspace activity — every record you anchor, DMSMS case you track, and readiness calculation you run contributes to your score. It outputs a letter grade (A+ through F) for executive reporting.
What are the pricing tiers? +
S4 Ledger offers four tiers: Pilot (Free during beta — 1,000 anchors/month, SDK access), Professional ($499/month — 10,000 anchors, full API, priority support), Enterprise ($2,499/month — 100,000 anchors, dedicated support, SLA), and Government ($4,999/month — unlimited anchors, FedRAMP pathway, on-premises option, dedicated CSM). All tiers include the full 19-tool ILS Workspace and SDK Playground.
How does the AI Supply Chain Risk Engine work? +
The risk engine analyzes supply chain health across 500+ defense platforms using ML-powered scoring. It evaluates supplier health indicators including GIDEP alerts, DLA lead time spikes, financial distress signals, single-source dependency, and counterfeit risk indicators. Each component receives a risk level (Critical/High/Medium/Low) with ETA impact calculations, sourced from 37 real Tier 1/1.5/2 defense suppliers.
Is our actual data safe? +
Yes. S4 Ledger processes everything client-side in your browser. No documents, part numbers, serial numbers, or sensitive information are ever sent to our servers or the blockchain. The only thing that leaves your browser is a SHA-256 hash — a 64-character mathematical fingerprint. Even if someone intercepted the hash, they cannot reverse-engineer any data from it.
Can I export or download my data? +
Yes. The Audit Record Vault stores every anchored record locally and supports CSV and XLSX export. The Compliance Scorecard exports scorecard reports. The Audit Report Generator produces multi-section audit packages. Transaction logs support CSV export with platform, date, and type filters. Calendar events export to .ics format.
What programming languages does the SDK support? +
The current SDK is Python-based with 27 functions and a 12-command CLI. It's pip-installable and works with Python 3.7+. JavaScript, Go, and Rust SDKs are planned for Phase 3. In the meantime, any language that can make HTTP requests can use our REST API directly — the 7 API endpoints accept standard JSON.
How much does each anchor transaction cost? +
Each anchor transaction costs approximately $0.01 in $SLS tokens. The underlying XRPL transaction fee is ~$0.001. Organizations can pay in USD — our fiat conversion mode handles the $SLS acquisition automatically, so users never need to interact with cryptocurrency. Batch anchoring supports up to 1,000 records per batch for high-volume operations.
